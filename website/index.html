How it works

  1. 01

    Observe

    The VS Code extension tracks file edits, commands, and prompts during a session.

  2. 02

    Extract

    After a successful fix, an agent extracts a named skill with triggers and steps.

  3. 03

    Store

    Skills are embedded, scored, and promoted when confidence is high enough.

  4. 04

    Reuse

    On the next similar task, TokenOS matches the skill and returns a compressed prompt.

Connect to your IDE

The API is hosted on this site — no local backend needed. Point your extension or MCP client at this URL.

1. Add API keys on Vercel

In your Vercel project → Settings → Environment Variables, add:

GOOGLE_API_KEY=your_key
GROQ_API_KEY=your_key          # optional fallback
GEMINI_MODEL=gemini-flash-latest

Redeploy after adding keys. Check: /health

2. Install the extension

git clone <your-repo-url> Tokens_AI
cd Tokens_AI/extension
npm install && npm run compile

Open extension/ in VS Code → press F5.

3. Point extension at this deployment

VS Code → Settings → search tokenos.apiUrl — set to your hosted URL:

https://your-app.vercel.app
  • TokenOS: AI Optimize Task — get a compressed prompt
  • TokenOS: Simulate Developer Workflow — seed demo skills
  • TokenOS: Open Dashboard — view savings

Same extension in Cursor. Add MCP config below for agent-native optimization.

4. MCP (Cursor / Claude)

cd mcp-server && pip install -r requirements.txt

Add to .cursor/mcp.json:

{
  "mcpServers": {
    "tokenos": {
      "command": "python",
      "args": ["/absolute/path/to/Tokens_AI/mcp-server/server.py"],
      "env": {
        "TOKENOS_API_URL": "https://your-app.vercel.app/api"
      }
    }
  }
}
